# Модули для системы сборки

Этот каталог позволяет разделить функциональность сборки на отдельные файлы `*.cmake`, которые необходимо включить в основной файл `CMakeLists.txt`, расположенный в корне проекта.

## Содержание

### Основные

* `compile_definitions.cmake` - конфигурация и проброс определений для целевой платформы.
* `environment_paths.cmake` - содержит основные пути к каталогам библиотеки.

### Дополнительные

* `toggle_options.cmake` - опции для сборки и управления параметрами библиотеки.
* `collect_options_to_compiler_definitions.cmake` - получает все переменные (опции) и добавляет их в список определений компиляции, если они активированы.
* `collect_source_files.cmake` - автоматически собирает все исходные файлы `*.c` из директории `src` и заносит их в переменную.
* `generate_doxygen.cmake` - добавляет цель для генерации `Doxygen` документации.
* `generate_numeric_limits.cmake` - генерация числовых ограничений для типов данных библиотеки.
* `parse_version.cmake` - разбирает версию из `CMake` на компоненты и пробрасывает в библиотеку.

## Описание

### Определения компилятора

Файл `compile_definitions.cmake` отвечает за инициализацию переменной `AE_TARGET_COMPILE_DEFINITIONS`, которая впоследствии используется в функции `target_compile_definitions`. Важно правильно расположить этот файл в основном `CMakeLists.txt`, поскольку другие скрипты могут добавлять дополнительные определения в эту переменную в процессе выполнения. Основная цель данного подхода заключается в том, чтобы обеспечить возможность модульного дополнения определений, не влияя на другие части проекта. Это позволяет сохранять гибкость и управляемость конфигурации сборки, облегчая добавление новых параметров без риска нарушить существующие настройки.

### Пути окружения

Файл `environment_paths.cmake` выполняет схожую функцию с `compile_definitions.cmake`, но его основное предназначение заключается в хранении ключевых путей, по которым будут искаться заголовочные файлы, исходные файлы, библиотеки и другие ресурсы проекта. Правильное расположение этого файла также имеет значение, и рекомендуется подключать его после `compile_definitions.cmake`. Это обеспечивает корректную настройку среды сборки и минимизирует вероятность возникновения конфликтов или ошибок, связанных с путями к ресурсам. Такой подход способствует более организованному управлению зависимостями проекта и упрощает процесс разработки, особенно в крупных командах или сложных проектах.

### Переключение опций

Файл `toggle_options.cmake` предоставляет возможность изменять настройки сборки и управлять параметрами библиотеки. С его помощью разработчики могут легко активировать или деактивировать различные опции, что позволяет настраивать поведение библиотеки в соответствии с конкретными требованиями проекта.

### Извлечение версии

Файл `parse_version.cmake` отвечает за извлечение версии библиотеки из файла `CMakeLists.txt`, разделяя её на составляющие: `AE_VERSION_MAJOR`, `AE_VERSION_MINOR` и `AE_VERSION_PATCH`. Эти значения затем передаются в библиотеку, что позволяет обеспечить правильное управление версиями в процессе сборки. Такой подход не только упрощает отслеживание изменений и обновлений в библиотеке, но и помогает избежать конфликтов, связанных с несовместимостью версий.

### Сбор исходных файлов

Файл `collect_source_files.cmake` автоматизирует процесс сбора всех исходных файлов, находящихся в директории, указанной в переменной `AE_TARGET_SOURCE_DIRECTORY`, и добавляет их в переменную `AE_TARGET_SOURCE_FILES`. Этот механизм упрощает управление исходным кодом, позволяя разработчикам легко включать новые файлы в проект без необходимости вручную обновлять список исходников.

### Сбор активированных опций

Файл `collect_options_to_compiler_definitions.cmake` представляет собой скрипт, который собирает все опции. Он проверяет каждую из этих переменных на соответствие заданному шаблону и добавляет их в список определений компиляции, если они активированы. Такой подход позволяет централизованно управлять настройками компиляции, упрощая процесс конфигурации проекта.

### Генерация лимитов целочисленных типов

Файл `generate_numeric_limits.cmake` предназначен для автоматической генерации ограничений для стандартных целочисленных типов данных библиотеки, адаптированных под конкретную целевую платформу. В процессе работы он опирается на заголовочный файл `numeric_limit.h`, который содержит набор макросов, позволяющих определить минимальные и максимальные значения для каждого типа, описанного в `numeric_types.h`. 

### Генерация документации

Файл `generate_doxygen.cmake` создает новую сборочную цель, которая позволяет автоматически генерировать документацию с помощью `Doxygen`, основываясь на уже имеющемся коде проекта. Этот файл упрощает процесс документирования, обеспечивая интеграцию с системой сборки и позволяя разработчикам легко обновлять документацию при внесении изменений в код.

## Рекомендации

* Используйте `snake_case` в имени файла, чтобы отразить его основное назначение.
* При включении модуля в основной файл `CMakeLists.txt` укажите краткое описание в виде комментария над ним.
